# 12-Layer Mapping

| Layer | Package(s) | Current Phase 7B Status |
| --- | --- | --- |
| Goal Layer | `internal/taskspec` | `TaskSpec` remains the only executable goal contract. Life events enter through `life_event_trigger`, behavior now enters through deterministic `behavior_intervention` intake, generated downstream tasks stay `TaskSpec + generation metadata`, and child workflows still consume standard `TaskSpec` instead of a second goal type. |
| Observation Layer | `internal/observation`, `internal/tools` | Ledger/document adapters remain live, and Phase 4A adds concrete `event source` and `calendar/deadline source` adapters. Workflow C no longer consumes raw event text; it consumes typed event/deadline evidence with provenance, confidence, source, and time range. |
| State Layer | `internal/state`, `internal/reducers` | Reducers still build deterministic `EvidencePatch` and apply snapshot/diff/versioned updates. Workflow C now uses the same reducer path to turn event evidence into a state diff instead of inventing a side channel for life events. |
| Context Engineering Layer | `internal/context`, `internal/prompt`, `internal/memory` | Planning / execution / verification context is token-aware, and the behavior path now enriches execution context with selected skill family/version/recipe plus procedural-memory references before dispatch. |
| Memory Layer | `internal/memory` | Memory is now a real substrate: durable SQLite-backed records/relations/embeddings/terms/audit/write-events, provider-backed embeddings, hybrid retrieval, configurable rejection policy, and cross-session influence on Monthly Review planning/cashflow reasoning. Phase 6B extends the same substrate with procedural skill-outcome memory that can change later skill selection. Runtime durable store remains a separate semantic plane. |
| Planning / Policy Layer | `internal/planning`, `internal/agents`, `internal/taskspec`, `internal/model`, `internal/structured`, `internal/skills` | `PlannerAgent` still owns the provider-backed planning path for Monthly Review, while the 6B behavior path adds skill-aware block planning. Planner declares allowed skill families and selection hints; orchestrator-side selection resolves concrete family/version/recipe before execution so `plan.Blocks` stays the execution truth source. |
| Skills + Tools Layer | `internal/skills`, `internal/tools`, `internal/analysis`, `internal/agents`, `internal/model`, `internal/finance`, `internal/behavior` | Skills are now first-class system capabilities with manifests, family/version/recipe metadata, policy, typed selection reasons, and runtime execution records. `internal/behavior` is now a formal deterministic domain with metrics/anomalies/recommendations/validators. `internal/finance` remains the live path's numeric truth source; behavior does not replace cashflow/debt/tax/portfolio. |
| Runtime / Harness Layer | `internal/runtime`, `internal/app`, `cmd/worker`, `internal/eval`, `cmd/eval`, `cmd/replay`, `internal/artifacts`, `cmd/api` | Runtime remains the same closeout-hardened async backbone from 7A, while 7B productizes it through API-served operator UI, profile-aware launch scripts, deterministic demo seeding, and operator-visible profile metadata. |
| Protocol Layer | `internal/protocol`, `internal/agents`, `internal/runtime`, `internal/structured`, `internal/api`, `internal/product`, `schemas/public/v1` | Protocol remains execution-first internally, but 7B adds a versioned external exposure seam. `/api/v1` is now the canonical operator/read surface, public schema docs live under `schemas/public/v1`, and `AgentEnvelope` is documented only as reference/internal execution protocol exposure rather than a public write contract. |
| Verification Layer | `internal/verification`, `internal/agents`, `internal/structured`, `internal/finance`, `internal/behavior` | Verification now goes beyond schema and narrow grounding. 5D adds deterministic grounding validators, numeric consistency validators, and finance business-rule validators. 6B adds behavior-specific evidence/anomaly/recommendation consistency checks so behavior interventions cannot bypass the same verification backbone. |
| Safety / Governance / HITL Layer | `internal/governance`, `internal/agents`, `internal/runtime`, `cmd/api` | Governance now consumes typed recommendation type + risk level + disclosure state and can deterministically move Debt vs Invest into `waiting_approval`. Phase 6B extends the same governance plane so `discretionary_guardrail / hard_cap.v1` also enters `waiting_approval`. Runtime mapping is still fixed: validator fail -> `failed`, `RequireApproval` -> `waiting_approval`, `Deny` -> `failed(governance_denied)`, operator approval -> resume continuation. |
| Observability / Evaluation / Feedback Layer | `internal/observability`, `internal/runtime`, `internal/api`, `internal/product`, `internal/eval`, `cmd/eval`, `cmd/replay`, `web` | 6A makes replay/eval/debug load-bearing, 6B extends it for skill/behavior attribution, 7A adds async runtime explanation, and 7B turns those same truth surfaces into operator-usable UI, benchmark list/detail/compare/export endpoints, and operator-facing replay/artifact views without creating a second observability truth source. |

## Trade-Offs

- The repository should now be described as **a governed, verifiable, observable agent system with a usable operator surface, deployment profiles, versioned external legibility, and formal benchmark/reporting outputs on top of a closeout-hardened async runtime backbone**.
- `CashflowAgent` and `DebtAgent` remain the load-bearing domain agents for Monthly Review and Debt vs Invest.
- `PlannerAgent` and `CashflowAgent` are now the only agents on the real provider-backed intelligence path in this phase, and only within Monthly Review.
- `DebtAgent` remains deterministic; 5B should not be narrated as if the whole system is already intelligence-promoted.
- Finance Engine is now the current live path's numeric truth source; models can explain numbers but should not originate numeric truth.
- `TaxAgent` and `PortfolioAgent` are now real execution boundaries, but only inside Workflow C for controlled scope.
- `TaskGenerationAgent` still only generates typed downstream tasks; runtime now executes a narrow allowlisted subset instead of turning task generation into recursive orchestration.
- `LifeEventAssessmentReport` remains a secondary artifact contract so Workflow C stays protocol-complete without turning the workflow back into a report-first system.
- `TaxOptimizationWorkflow` and `PortfolioRebalanceWorkflow` are the first two capability-backed child workflows, and the only auto-executable follow-up intents in the current phase.
- `BehaviorInterventionWorkflow` is now the first formal behavior-domain workflow, but it intentionally remains separate from Workflow C auto-generation and does not replace Monthly Review or Debt vs Invest.
- runtime promotion now hardens the same loop with durable queue/lease semantics, scheduler-generated wakeups, fencing-token safety, promoted-backend proofs, and a real Postgres promotion profile instead of leaving runtime truth locked to SQLite-only assumptions.
- 7B productization does not create a second control plane: UI, `/api/v1`, profile docs, and benchmark/reporting surfaces all sit on top of the same typed runtime/operator/replay/eval services.
- replay truth source is now runtime durable truth + replay/debug projections + artifact refs, not artifacts-only and not a separate explanation database.
- canonical 6A regression is deterministic/mock only; live provider paths remain smoke/manual evidence.
- canonical 6B regression is also deterministic/mock only and lives in a separate `phase6b-default` corpus so 6A semantics stay stable.
- live Temporal, remote agentization, public writable protocol servers, pgvector-backed memory infra, fuller artifact lifecycle management, and full external observability infra remain deferred behind stable interfaces.
