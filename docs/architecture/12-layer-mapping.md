# 12-Layer Mapping

| Layer | Package(s) | Current Phase 5C Status |
| --- | --- | --- |
| Goal Layer | `internal/taskspec` | `TaskSpec` remains the only executable goal contract. Life events enter through `life_event_trigger`, generated downstream tasks stay `TaskSpec + generation metadata`, and follow-up child workflows still consume standard `TaskSpec` instead of a second goal type. |
| Observation Layer | `internal/observation`, `internal/tools` | Ledger/document adapters remain live, and Phase 4A adds concrete `event source` and `calendar/deadline source` adapters. Workflow C no longer consumes raw event text; it consumes typed event/deadline evidence with provenance, confidence, source, and time range. |
| State Layer | `internal/state`, `internal/reducers` | Reducers still build deterministic `EvidencePatch` and apply snapshot/diff/versioned updates. Workflow C now uses the same reducer path to turn event evidence into a state diff instead of inventing a side channel for life events. |
| Context Engineering Layer | `internal/context`, `internal/prompt`, `internal/memory` | Planning / execution / verification context is token-aware, and 5C now adds formal planner/cashflow memory query builders so retrieval query formation itself becomes a typed context/memory concern instead of workflow-local string glue. |
| Memory Layer | `internal/memory` | Memory is now a real substrate: durable SQLite-backed records/relations/embeddings/terms/audit/write-events, provider-backed embeddings, hybrid retrieval, configurable rejection policy, and cross-session influence on Monthly Review planning/cashflow reasoning. Runtime durable store remains a separate semantic plane. |
| Planning / Policy Layer | `internal/planning`, `internal/agents`, `internal/taskspec`, `internal/model`, `internal/structured` | `PlannerAgent` now has a real provider-backed structured planning path for Monthly Review. It still compiles back into the same typed `ExecutionPlan`, so `plan.Blocks` remains the only execution truth source. `TaskGenerationAgent` still generates downstream tasks without becoming a coordinator or executor. |
| Skills + Tools Layer | `internal/tools`, `internal/analysis`, `internal/agents`, `internal/model` | Deterministic tools remain the execution substrate and still own numeric finance truth. `CashflowAgent` now layers provider-backed structured reasoning on top of deterministic metrics, while `DebtAgent`, `TaxAgent`, and `PortfolioAgent` remain deterministic or deferred according to previous phase boundaries. |
| Runtime / Harness Layer | `internal/runtime`, `internal/app`, `cmd/worker` | Local Temporal-aligned runtime now manages follow-up task graphs, capability activation, task state advancement, child workflow dispatch, resumable execution records, committed state handoff, retry metadata, suppression/blocking reasons, and durable SQLite-backed persistence for graphs/executions/checkpoints/approvals/replay. Worker passes can reevaluate and resume tasks outside the parent workflow. |
| Protocol Layer | `internal/protocol`, `internal/agents`, `internal/runtime`, `internal/structured` | Protocol remains execution-first and is only extended, not replaced. 5B adds structured-output schemas and trace metadata without inventing a second backbone. `TaskSpec`, `ExecutionPlan`, `AgentEnvelope`, and `SystemStepBus` stay additive-only. |
| Verification Layer | `internal/verification`, `internal/agents`, `internal/structured` | Verification still covers block + final validation and follow-up child workflows. 5B adds schema validation, deterministic grounding pre-checks, metric/evidence ref consistency, caveat/risk-flag presence checks, and a narrow numeric-consistency guard for provider-backed planner/cashflow paths, but does not expand into LLM-as-judge or deeper finance hardening. |
| Safety / Governance / HITL Layer | `internal/governance`, `internal/agents`, `internal/runtime`, `cmd/api` | Governance continues to control approval/disclosure, and runtime now turns approval state, deny semantics, resume anchors, operator actions, and idempotent transitions into a real operator plane rather than workflow-local fields. |
| Observability / Evaluation / Feedback Layer | `internal/observability`, `internal/runtime`, `internal/api`, `cmd/eval` | Replay/trace still spans parent Workflow C -> generated task -> child workflow -> child artifact -> child state commit, and 5C additively extends Monthly Review evidence with memory query, hit/reject/select traces plus embedding call/usage evidence. Operator-facing durable replay remains sourced from `ReplayStore`; memory/intelligence traces are still additive workflow evidence rather than a new durable operator truth plane. |

## Trade-Offs

- The repository should now be described as **system-agent backbone + first real domain-agent execution path + first proactive life-event loop + first capability-backed follow-up execution + first operator-runnable durable runtime plane + real-intelligence-backed Monthly Review golden path + first real memory substrate**.
- `CashflowAgent` and `DebtAgent` remain the load-bearing domain agents for Monthly Review and Debt vs Invest.
- `PlannerAgent` and `CashflowAgent` are now the only agents on the real provider-backed intelligence path in this phase, and only within Monthly Review.
- `DebtAgent` remains deterministic; 5B should not be narrated as if the whole system is already intelligence-promoted.
- `TaxAgent` and `PortfolioAgent` are now real execution boundaries, but only inside Workflow C for controlled scope.
- `TaskGenerationAgent` still only generates typed downstream tasks; runtime now executes a narrow allowlisted subset instead of turning task generation into recursive orchestration.
- `LifeEventAssessmentReport` remains a secondary artifact contract so Workflow C stays protocol-complete without turning the workflow back into a report-first system.
- `TaxOptimizationWorkflow` and `PortfolioRebalanceWorkflow` are the first two capability-backed child workflows, and the only auto-executable follow-up intents in the current phase.
- local durable persistence, worker reevaluation, and operator APIs are now the live bridge from code-level loop closure to runtime-level loop closure.
- Behavior-domain execution, full finance hardening, stronger remote memory infra, real Temporal/Postgres/pgvector/MinIO, and a fuller replay/eval plane remain deferred behind stable interfaces.
